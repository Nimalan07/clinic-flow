ClinicFlow
Smart Clinic Queue Management System
Hackathon Submission: Queue Cure '26
Developer: Nimalan Mani M
Framework: React.js, Node.js, Express
Database: Supabase (PostgreSQL)

1. Executive Summary

ClinicFlow is a real-time clinic queue management platform designed to replace traditional paper-token systems used in neighborhood clinics.

Patients often wait for long periods without knowing their queue status, while receptionists manually manage queues using paper slips and verbal announcements. This leads to confusion, inefficiency, and poor patient experience.

ClinicFlow digitizes the entire process by providing:

  • Instant patient registration
  • Automatic token generation
  • Live queue tracking
  • Real-time updates across devices
  • Estimated waiting times
  • QR-based patient access

The platform enables both receptionists and patients to view queue information without requiring page refreshes.

2. Problem Statement

A large percentage of small clinics still rely on paper tokens and manual queue management.

Common challenges include:

  • Patients do not know their position in the queue.
  • Receptionists manually track every patient.
  • Doctors have no visibility into queue progress.
  • Patients repeatedly ask receptionists for updates.
  • Waiting times are unpredictable.
  • No centralized queue dashboard exists.

As clinic traffic increases, manual systems become difficult to manage and create a poor patient experience. The challenge was to create a simple and affordable digital queue management system that works in real time.

3. Proposed Solution

ClinicFlow provides a centralized digital queue system that connects the clinic staff and patients seamlessly.

Receptionist Features

4. System Architecture

ClinicFlow follows a standard client-server architecture built on web sockets to deliver updates in sub-second timelines.

Frontend Stack

  • React.js (Single Page Application architecture)
  • Tailwind CSS (Premium layout design)

Backend Stack

  • Node.js (Runtime)
  • Express.js (Web API routing)

Real-Time Communication

  • Socket.IO (Full duplex TCP communication)

Database

  • Supabase PostgreSQL

Architecture Flow

Event Loop
Reception Dashboard → Socket.IO Events → Node.js Server → Supabase Database → Patient Display → Live Updates

All connected clients receive queue updates instantly whenever a change occurs in the database.

5. Database Design

The queue state is fully structured in a PostgreSQL database table named patients.

Field Type Description
id bigint (PK) Unique auto-incremented record identifier.
token integer Patient sequence number.
name text Name of the patient.
status text State of patient: waiting, serving, completed.
joined_at timestamp Timestamp when the patient was registered.
start_time timestamp Timestamp when called to the doctor's room.
end_time timestamp Timestamp when consultation completed.
consultation_duration integer Elapsed time in minutes.

6. Socket.IO Event Flow

The real-time coordination of the application uses custom Socket.IO event payloads:

Event: addPatient
Receptionist adds patient → Server generates token → Database updated → queueUpdated broadcast → All clients receive update
Event: callNext
Receptionist clicks Call Next → Current patient marked completed → Next patient marked serving → Database updated → queueUpdated broadcast → Patient display updates instantly
Event: resetQueue
Receptionist clicks Reset Queue → All records cleared → Database reset → queueUpdated broadcast → All screens refreshed automatically

8. Key Features

  • Real-Time Queue Synchronization: Changes appear instantly across all devices using WebSockets.
  • Automatic Token Generation: Patients receive consecutive numerical tokens automatically.
  • QR-Based Access: Waiting patients scan a live QR card to monitor the queue from their mobile device.
  • Estimated Waiting Time: Calculated dynamically based on actual finished consultation duration metrics.
  • Duplicate Prevention: Backend lock filters rapid receptionist double-clicks to protect queue order.
  • Persistent Storage: Database holds active queue logs, allowing restart recovery.

10. Results and Outcomes

ClinicFlow successfully enables real-time queue synchronization between receptionist and patient screens without requiring page refreshes.

Using Socket.IO, patient registrations, token updates, and queue status changes are instantly reflected across all connected devices. The platform delivers instant queue visibility, dynamic estimated wait times, duplicate prevention, and easy mobile QR access. The solution demonstrates how real-time technologies can modernize clinic operations and improve patient experience.

11. Future Enhancements

Given additional development time, the platform can be expanded with:

  • Doctor Dashboard: Dedicated interface for updating consultation details.
  • Appointment Scheduling: Pre-booking online slots.
  • SMS & WhatsApp Notifications: Alerting patients when they are 2 tokens away.
  • Voice Token Announcements: Text-to-speech engine reading out token numbers.
  • Analytics Dashboard: Clinic workflow statistics and efficiency trends.
  • AI-powered Wait Predictions: Using machine learning to predict times based on symptoms/patient profiles.

12. Conclusion

ClinicFlow successfully addresses the challenges of manual clinic queue management by introducing a real-time, digital queue system.

By combining React.js, Socket.IO, Node.js, and Supabase, the platform delivers instant synchronization, improved patient transparency, and reduced receptionist workload. The project demonstrates how modern web technologies can solve everyday operational problems in healthcare environments while remaining simple, scalable, and cost-effective.
